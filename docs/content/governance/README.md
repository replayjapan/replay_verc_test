# Content Governance

Rules that control content quality and consistency across the site.
All content generators (Codex, Cowork, EngAI) must read these before creating content.

## Files

| File | Purpose |
|------|---------|
| `voice-brief.md` | Editorial voice and tone guidelines |
| `domain-copy-rubric.md` | Field-by-field writing rules for domain descriptions |
| `approved-facts.md` | Verified claims that can be used in content |
| `set-thesis-lines.md` | Per-set narrative direction |

## Rules

- PLAI drafts governance docs, Codex writes production copy
- Every content generation task must reference which governance files to read
- Developer approves all governance changes
